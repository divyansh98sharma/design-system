# AI Bubbles — Usage

A horizontal row of pill-shaped suggestion chips used to surface AI-generated
quick replies, follow-up prompts, or one-tap actions. The optional
hand-pointing leading icon hints that the chips are AI suggestions and helps
distinguish them from regular chip groups.

## When to use

- **AI quick replies** — model-generated follow-up questions or short reply suggestions.
- **AI-suggested actions** — one-tap actions surfaced by an AI assistant (e.g. "Order labs", "Summarise note").
- **Inline prompts** — short, alternative inputs the user can pick instead of typing.

## When **not** to use

- **Status / tags** — use the [`Chip`](/?path=/docs/components-chip--docs) component, which has explicit colour states (`active`, `error`, `warning`, etc.).
- **Filters** — use a filter chip group with selected/unselected states.
- **Navigation tabs** — use [`Tabs`](/?path=/docs/components-tabs--docs).
- **More than 5 chips on one line** — wrap, paginate, or move overflow into a "More" affordance. The Figma library defines variants only up to 5.

## Anatomy

```
[ leading-icon ]   [ chip₁ ]   [ chip₂ ]   …   [ chipₙ ]
       │              │
       │              └── pill-shaped button: optional icon + label
       └── hand-pointing cursor (defaults on, can be hidden)
```

| Slot           | Size         | Notes                                                              |
| -------------- | ------------ | ------------------------------------------------------------------ |
| Leading icon   | 16 × 16      | Hand-pointing cursor by default; override via `leadingIconPath`.   |
| Chip           | 20 px tall   | Pill (`border-radius: 999px`), 1 px gradient border (teal → dark teal → terracotta). |
| Chip icon      | 16 × 16      | Optional per chip via `chip.iconPath`.                             |
| Chip label     | 12 / 16      | Open Sans regular, single line, no wrap.                           |
| Container gap  | 8 px         | Between leading icon and chips, and between chips.                 |

## Variants

Five Figma variants — **One Chip → Five Chips** — control how many chips
render in the row. Pass an array of `AiBubbleChip` objects of any length;
the component renders one chip per entry.

```ts
interface AiBubbleChip {
  label: string;
  iconPath?: string | null; // optional 24×24 SVG path
}
```

## Do / don't

**Do**
- Keep labels short — 1 to 3 words. Long labels look awkward in a 20 px pill.
- Group chips that share a single underlying intent (e.g. all "next questions").
- Emit a single `chipSelected` handler and switch on `chip.label` or an enum.

**Don't**
- Don't mix this component with status chips or filter chips on the same row — visual noise.
- Don't disable individual chips. If a suggestion is unavailable, omit it from the array instead.
- Don't change the border colour per chip to convey state. Use [`Chip`](/?path=/docs/components-chip--docs) for that.

## Accessibility

- Each chip is a real `<button type="button">`, keyboard-focusable and Enter/Space activatable.
- The row has `role="group"` with an `aria-label` (`"AI suggestions"` by default; override via `ariaLabel`).
- The leading icon is decorative (`aria-hidden="true"`); the visible label carries meaning.
- Focus state: 2 px outline in `#007B95` (the first gradient stop) with a 2 px offset — meets WCAG 2.1 SC 2.4.7 (Focus Visible).
- Label contrast against the white chip background uses `--color-text-primary` on `--color-surface`, well above WCAG AA (4.5:1) for normal text.
- The 1 px gradient border is decorative — the text label, not the border colour, carries meaning. Consumers relying on screen readers or high-contrast mode are unaffected.
- Hover background uses `--color-user-5` (`#F1FBFF`); the gradient border remains visible on hover.

defined in `src/ds/tokens/_component.scss`. Override these to retheme without
forking the component.
