## What is it?

A button triggers an action when clicked. It should always be used for **actions** (save, submit, delete, open). Never use a button for navigation — use a link instead.

<div className="bg-do-dont">
  <div className="bg-do">
    <h4>✅ Use a button when…</h4>
    <p>The user is performing an action: Submit form, Save changes, Delete record, Open modal, Download file</p>
  </div>
  <div className="bg-dont">
    <h4>❌ Don't use a button when…</h4>
    <p>The user is navigating to another page or URL — use an anchor tag or router link instead.</p>
  </div>
</div>

### Type hierarchy rule

```
Primary  →  most important action (one only)
Secondary  →  supporting actions (multiple ok)
White  →  on coloured backgrounds
Error  →  permanent / destructive (always with a confirmation)
Warning  →  risky but reversible
Disabled  →  action not available (auto-applied via [disabled]="true")
```

## Styles

The <code>btnStyle</code> prop controls the button's shape.

<table className="bg-table">
  <thead>
    <tr><th>Style</th><th>Input value</th><th>What it does</th><th>When to use</th></tr>
  </thead>
  <tbody>
    <tr>
      <td>Default</td>
      <td><code>btnStyle="default"</code></td>
      <td>Single button, single click target</td>
      <td>Most cases</td>
    </tr>
    <tr>
      <td>Divided (split button)</td>
      <td><code>btnStyle="divided"</code></td>
      <td>Left section = main action, right section = chevron/secondary action</td>
      <td>When one button needs two related actions (e.g. "Save" + "Save & Close")</td>
    </tr>
  </tbody>
</table>

The divided style emits two separate events: `buttonClick` (left section) and `dividerClick` (right section).

## Usage examples

### Minimal
```html
<ds-button label="Save"></ds-button>
```

### Destructive action with error type
```html
<ds-button
  label="Delete Patient"
  type="error"
  (buttonClick)="onDelete()">
</ds-button>
```

### Icon + label
```html
<ds-button
  label="Download"
  type="secondary"
  [icon]="downloadSvg"
  (buttonClick)="onDownload()">
</ds-button>
```

### Icon-only (must have aria-label)
```html
<ds-button
  [icon]="settingsSvg"
  [iconOnly]="true"
  type="secondary"
  aria-label="Open settings"
  (buttonClick)="openSettings()">
</ds-button>
```

### Divided split button
```html
<ds-button
  label="Save"
  type="primary"
  btnStyle="divided"
  (buttonClick)="onSave()"
  (dividerClick)="openSaveMenu()">
</ds-button>
```

### With counter
```html
<ds-button
  label="Inbox"
  type="secondary"
  [counter]="unreadCount">
</ds-button>
```

### Disabled
```html
<ds-button
  label="Submit"
  type="primary"
  [disabled]="form.invalid">
</ds-button>
```

## States

| State | How to trigger | Visual |
|-------|---------------|--------|
| Default | Normal render | Base background and border |
| Hover | Mouse over (not disabled) | Slightly darker background |
| Disabled | `[disabled]="true"` | Gray background `#f7f7f7`, gray border `#ebebeb`, gray text `#717171`, cursor: not-allowed |
| Focus | Tab key or click-and-hold | 2px outline using `--button-focus-color` |
| Active | Mouse down | Browser default or same as hover |

The **disabled state cannot be hovered** — pointer-events are disabled. Do not rely on hover for disabled buttons in tests.

## Accessibility

<table className="bg-table">
  <thead>
    <tr><th>Requirement</th><th>Why</th><th>How</th></tr>
  </thead>
  <tbody>
    <tr>
      <td>Icon-only buttons must have a label</td>
      <td>Screen readers cannot describe an SVG icon</td>
      <td>Add <code>aria-label="Description"</code> on <code>&lt;ds-button&gt;</code></td>
    </tr>
    <tr>
      <td>Disabled must use <code>[disabled]</code> input</td>
      <td>Native <code>disabled</code> attribute is needed for assistive tech</td>
      <td>Use <code>[disabled]="true"</code>, not just CSS opacity</td>
    </tr>
    <tr>
      <td>Error buttons need confirmation</td>
      <td>Irreversible actions must be confirmed before executing</td>
      <td>Show a confirm modal or inline confirmation step</td>
    </tr>
    <tr>
      <td>Focus is always visible</td>
      <td>Keyboard users must see where focus is</td>
      <td>Built-in — do not override <code>outline: none</code> on <code>:focus-visible</code></td>
    </tr>
    <tr>
      <td>Alert indicator needs hidden text</td>
      <td>The red dot has no text — screen readers skip it</td>
      <td>Add <code>&lt;span class="sr-only"&gt;Has notifications&lt;/span&gt;</code> near the button</td>
    </tr>
  </tbody>
</table>

## QA checklist

When testing any screen that contains a button:

- [ ] Each page/modal has at most **one** primary button
- [ ] Clicking a disabled button triggers **no** action and emits **no** event
- [ ] Icon-only buttons have a visible tooltip or `aria-label`
- [ ] Error-type buttons open a confirmation dialog before executing the action
- [ ] Tab key focuses the button and shows the focus ring
- [ ] Counter chip displays the correct number from the data source
- [ ] Alert indicator dot appears only when the condition is true
- [ ] Divided button: left click → `buttonClick` event, right click → `dividerClick` event
