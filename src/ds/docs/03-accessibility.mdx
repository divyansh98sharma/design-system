### Focus indicators

<p>
  Every focusable element gets a <code>2px solid var(--button-focus-color)</code> outline
  with a 2px offset when <code>:focus-visible</code> is active. This uses the CSS
  <code>:focus-visible</code> pseudo-class so the ring only appears during keyboard
  navigation, not on mouse click.
</p>

```scss
/* Standard focus pattern used across all components */
&:focus-visible {
  outline: 2px solid var(--button-focus-color);
  outline-offset: 2px;
}
```

<div className="a11y-callout">
  <strong>Never remove focus outlines</strong> without a better replacement.
  <code>outline: none</code> without an alternative is a WCAG 2.4.7 violation.
</div>

</div>

## ARIA patterns

<p>
  Angular's ARIA bindings should reflect the component's current state dynamically.
  Static <code>aria-*</code> attributes in templates are not enough when state changes.
</p>

### Required ARIA on common patterns

<div className="a11y-grid">
  <div className="a11y-card">
    <h4>Toggle / Switch</h4>
    <p>Use <code>role="switch"</code> with <code>aria-checked</code> bound to the toggle state.</p>
  </div>
  <div className="a11y-card">
    <h4>Icon-only buttons</h4>
    <p>Always provide <code>aria-label</code> describing the action, e.g. <code>"Close dialog"</code>.</p>
  </div>
  <div className="a11y-card">
    <h4>Modals / Dialogs</h4>
    <p>Use <code>role="dialog"</code>, <code>aria-modal="true"</code>, and <code>aria-labelledby</code> pointing to the title.</p>
  </div>
  <div className="a11y-card">
    <h4>Status messages</h4>
    <p>Toast / notification updates should use <code>role="status"</code> (polite) or <code>role="alert"</code> (assertive).</p>
  </div>
  <div className="a11y-card">
    <h4>Loading states</h4>
    <p>Loaders should have <code>role="status"</code> and <code>aria-label="Loading"</code>.</p>
  </div>
  <div className="a11y-card">
    <h4>Expandable sections</h4>
    <p>The trigger button needs <code>aria-expanded</code> bound to the open/closed state.</p>
  </div>
</div>

### Example — icon-only button

```html
<!-- ✓ Correct: aria-label on icon-only button -->
<ds-button
  [icon]="closeSvg"
  [iconOnly]="true"
  aria-label="Close dialog"
></ds-button>
```

```html
<!-- ✗ Wrong: no label — screen reader announces nothing useful -->
<ds-button
  [icon]="closeSvg"
  [iconOnly]="true"
></ds-button>
```

## Content and language

<p>
  Accessibility extends beyond interaction — the words on screen matter too.
</p>

### Error messages

<p>
  Error messages must be descriptive enough to act on without support.
</p>

| ✗ Don't | ✓ Do |
|---------|------|
| "Invalid input" | "Date of birth must be in MM/DD/YYYY format" |
| "Error" | "Password must be at least 8 characters" |
| "Something went wrong" | "Unable to save. Check your connection and try again." |

### Loading and async states

<p>
  Always communicate async state to screen readers using live regions:
</p>

```html
<!-- Polite: announced when the user is idle -->
<div role="status" aria-live="polite">
  Saving changes…
</div>

<!-- Assertive: interrupts immediately — use sparingly -->
<div role="alert" aria-live="assertive">
  Session will expire in 2 minutes.
</div>
```
