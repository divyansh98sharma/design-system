  Before a new component PR is ready for review, every item must be checked:
</p>

  </li>

    <div className="co-check">✓</div>

  </li>
  <li>
    <div className="co-check">✓</div>
    Component tokens are added to <code>_component.scss</code> under the component's section
  </li>
  <li>
    <div className="co-check">✓</div>
    No hardcoded hex colours or pixel values — all from tokens
  </li>
  <li>
    <div className="co-check">✓</div>

  </li>
  <li>
    <div className="co-check">✓</div>

  </li>
  <li>
    <div className="co-check">✓</div>

  </li>

## Component file structure

<p>
  Every component lives in its own directory under <code>src/ds/</code>:
</p>

```
src/ds/my-component/
├── my-component.component.ts      # Angular component
├── my-component.component.html    # Template
├── my-component.component.scss    # Scoped styles (uses component tokens)
└── my-component.stories.ts        # Storybook stories
```

### Story structure

<p>
  Stories follow this order:
</p>

```typescript
export const Playground: Story = { name: 'Playground' };

// All type/variant stories
export const Types: Story = { ... };
export const Sizes: Story = { ... };

// Specific feature stories
export const WithIcons: Story = { ... };

// State stories
export const Disabled: Story = { ... };

// Interaction tests (play functions)
export const ClickEmitsEvent: Story = {
  name: 'Interaction: Click emits event',
  play: async ({ canvasElement, args }) => { ... },
};
```

## Token conventions

### Naming pattern

```scss
/* component-scope — property — variant — state */
--button-primary-bg
--button-primary-bg-hover
--button-primary-border
--button-counter-text
--chip-ai-border-color
```

### Adding component tokens

<p>
  Add new tokens to the appropriate block in <code>src/ds/tokens/_component.scss</code>.
  Each component has its own clearly-labelled section. Never use a primitive token value
  as a component token default — always alias through the semantic layer.
</p>

```scss
// ─── My Component ───────────────────────────────────────────────────────────
--my-component-bg:        var(--color-surface-default);
--my-component-bg-hover:  var(--color-surface-hover);
--my-component-border:    var(--color-border-default);
--my-component-text:      var(--color-text-primary);
```

## Branch naming

| Type | Pattern | Example |
|------|---------|---------|
| New component | `feat/component-name` | `feat/date-picker` |
| Bug fix | `fix/description` | `fix/button-focus-ring` |
| Token update | `chore/token-name` | `chore/palette-tokens` |
| Documentation | `docs/section-name` | `docs/storybook-sections` |
| CI / infra | `ci/description` | `ci/dependabot-auto-merge` |

<div className="co-callout">
  <strong>Figma branch:</strong> When a component is being actively redesigned in Figma,
  open a matching code branch at the same time. Both branches should reference the same
  Figma node ID in commit messages and PR descriptions for traceability.
</div>
