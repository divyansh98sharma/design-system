## How versions work

<p>
  Every component in the Storybook sidebar shows its version as a coloured tag
  (<code>v1</code> in blue, <code>v2</code> in green). The tag is set in the story's
  <code>tags</code> array and reflects the component's current design generation:
</p>

| Version | Meaning |
|---------|---------|
| **v1.0** | Initial stable release |
| **v1.x** | Minor update — token or style fix, no API change |
| **v2.0** | Major update — new Figma spec, visual redesign, or API addition |
| **v2.x** | Minor update on top of a v2 major |

<div className="cl-callout">
  <strong>When to bump the version:</strong> increment the minor (v1.0 → v1.1) for token fixes and
  style-only changes. Increment the major (v1.x → v2.0) when the Figma spec is redesigned, when
  new <code>@Input</code> properties are added, or when visual changes are large enough to require
  Chromatic snapshot review.
</div>
