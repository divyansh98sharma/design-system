## Deprecating a component

### 1 — JSDoc in the component class

<p>
  Add <code>@deprecated</code> JSDoc to the component class. This causes IDEs to show a
  strikethrough and tooltip at every usage site — zero runtime cost.
</p>

<div className="dp-code">
{`/**
 * @deprecated since v2.1 — `}<span className="dp-string">{`use NewButtonComponent`}</span>{` instead.
 * Will be removed in v3.0.
 * Migration: replace <ds-old-button> with <ds-button>.
 */
@Component({ selector: `}<span className="dp-string">{`'ds-old-button'`}</span>{`, ... })
export class OldButtonComponent { }`}
</div>

### 2 — Story tag

<p>
  Add <code>'deprecated'</code> to the story's <code>tags</code> array.
  Storybook will show an amber banner on the canvas and a badge in the sidebar.
</p>

<div className="dp-code">
{`const meta: Meta<OldButtonComponent> = {
  title: `}<span className="dp-string">{`'Atoms/Old Button'`}</span>{`,
  component: OldButtonComponent,
  tags: [`}<span className="dp-string">{`'autodocs'`}</span>{`, `}<span className="dp-tag">{`'deprecated'`}</span>{`],
  parameters: {
    docs: {
      description: {
        component: `}<span className="dp-string">{`'**Deprecated since v2.1.** Use [Button](/story/atoms-button--playground) instead.'`}</span>{`,
      },
    },
  },
};`}
</div>

### 3 — Changelog entry

<p>
  Add a row to the component's <code>-changelog.mdx</code> documenting what changed,
  why it was deprecated, and the migration path.
</p>

</div>

## Deprecating a token

### 1 — Move to `_deprecated.scss`

<p>
  Cut the token from <code>_component.scss</code> or <code>_semantic.scss</code> and paste it
  into <code>src/ds/tokens/_deprecated.scss</code>. Point its value at the replacement token
  so existing consumers continue to compile.
</p>

<div className="dp-code">
{`/* src/ds/tokens/_deprecated.scss */

:root {
  `}<span className="dp-comment">{`/* @deprecated v2.1 — use --chip-bg-neutral */`}</span>{`
  --chip-bg-default: var(--chip-bg-neutral);
}`}
</div>

### 2 — Comment at the new token site

<p>
  Add a note at the new token definition so reviewers know an alias exists:
</p>

<div className="dp-code">
{`/* Replaces --chip-bg-default (deprecated v2.1, removed v3.0) */
--chip-bg-neutral: var(--color-surface-neutral);`}
</div>

<div className="dp-callout">
  <strong>Consumers who reference <code>var(--chip-bg-default)</code></strong> will still compile
  and render correctly until the major version removes the alias. They'll see the new value
  automatically.
</div>

</div>

## Deprecating a prop (`@Input`)

<p>
  When an input is renamed or replaced, keep the old one working as a pass-through
  and annotate it clearly:
</p>

<div className="dp-code">
{`/** @deprecated since v2.2 — use `}<span className="dp-string">{`btnStyle`}</span>{` */
@Input() set style(value: string) {
  this.btnStyle = value as ButtonStyle;
}`}
</div>

<p>
  This lets existing templates keep working without a hard error, while the IDE
  marks the old property as struck-through.
</p>

</div>

## Quick reference

| What you're deprecating | File to update | Tag / annotation |
|-------------------------|---------------|-----------------|
| Whole component | `component.ts` + `stories.ts` | `@deprecated` JSDoc + `'deprecated'` tag |
| A token | `_deprecated.scss` | `/* @deprecated vX.Y */` comment |
| A prop (`@Input`) | `component.ts` | `@deprecated` JSDoc on the setter |
| A story variant | `stories.ts` | `deprecated: true` in `parameters` |
